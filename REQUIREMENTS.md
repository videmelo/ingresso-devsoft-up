### GRUPO, APRESENTAÇÃO E TEMA
- O grupo deverá ser formado por no mínimo 4 e no máximo 5 alunos.
- A apresentação do trabalho será feita em grupo e a defesa do projeto será
feita de forma individual. O aluno que não fizer a defesa não receberá nota.
- O tema escolhido pela equipe deverá ser escolhido até a aula do dia 21 de
maio, para as turmas do turno da manhã, e dia 22 de maio, para as turmas
do turno da noite.
- Os grupos deverão passar os nomes para a representante da turma, junto
com o tema se já estiverem definidos. Como não poderá haver temas iguais,
será respeitada a ordem de envio para a representante. Após esse prazo,
não será possível trocar o tema, e as equipes que não estiverem definidas,
ficarão a cargo do professor a definição do tema.
- Lembrando que os temas, devem atender aos requisitos mínimos definidos.
CONTEXTO E OBJETIVO DO TRABALHO
- O trabalho deve ser desenvolvido em Java, utilizando os conceitos de programação
orientada a objetos que foram abordados durante o semestre, como tratamento de
exceções, persistência de dados, entre outros. Desta forma, deverá ser criado um
programa que resolva um problema prático aplicando estas habilidades em, por
exemplo, um sistema para uma papelaria, biblioteca, padaria, ou também um jogo
de cartas com complexidade razoável.
- Assim, para que não sejam criados sistemas simples, e incentivar a criatividade
garantindo um projeto mais robusto, considere no seu sistema o uso de 2 MVCs
por pessoa do grupo. Desta forma, cada pessoa do grupo criará duas classes,
com 2 views e 2 controllers no mínimo, reflitindo situações do mundo real.
- Por fim, lembre-se de que o trabalho entregue deve ser compatível com a carga
esperada para um grupo de 4 a 5 pessoas. E para que você não fique tão perdido,
é possível ver alguns exemplos e ideias nos próximos slides, bem como um exemplo
de como pedir ajuda de IA para colaborar com o desenvolvimento de sua ideia.

### REQUISITOS EM TÓPICOS - RESUMO
- Classes e Métodos Abstratos: Utilize classes e métodos abstratos onde for
apropriado.
- Herança: Deve haver pelo menos uma hierarquia de classes que demonstre herança.
- Polimorfismo: Demonstre o uso de polimorfismo em seu projeto.
- Interfaces: Implemente interfaces para definir contratos e promover polimorfismo.
- Tratamento de Exceções: Inclua tratamento adequado de exceções para garantir a
robustez da aplicação.
- Coleções: Utilize coleções para armazenar e manipular dados.
- Agregação, Composição e Associação: O projeto deve demonstrar o uso de
agregação, composição e associação.
- Persistência de Dados: Utilize leitura e gravação de dados em arquivos de texto ou
JSON. Considerando que a luz da sua barraquinha de cachorro quente pode
acabar e voltar em seguida e você não pode perder os pedidos ao religar o
sistema.

### REQUISITOS DETALHADOS
- Herança: O sistema deve apresentar uma superclasse que contenha atributos e métodos comuns a
diferentes entidades do projeto, promovendo a reutilização de código e a organização da
estrutura de classes.
- Classe Abstrata: É preciso implementar pelo menos uma classe abstrata que sirva como base para
outras classes. Essa classe abstrata deve conter pelo menos um método abstrato que será
implementado nas classes filhas.
- Polimorfismo de Sobrescrita: Deve-se utilizar o conceito de polimorfismo de sobrescrita, onde um
ou mais método da classe mãe é redefinido nas classes filhas para se adequar às suas
particularidades.
- Polimorfismo de Sobrecarga: Utilize o polimorfismo de sobrecarga ao implementar métodos com o
mesmo nome, mas com assinaturas diferentes, para lidar com situações específicas do projeto.
- Interface: O sistema deve conter pelo menos uma interface que defina um contrato a ser seguido
pelas classes que a implementarem. Essa interface pode estabelecer métodos obrigatórios ou
padrões a serem seguidos.
- CRUD's (Cadastro, Alteração, Deleção e Listagem): O sistema deve ter no mínimo 3 CRUD's,
permitindo o cadastro, alteração, deleção e listagem de informações relevantes para o contexto
do projeto.

### REQUISITOS DETALHADOS
- Encapsulamento: É fundamental aplicar os conceitos de encapsulamento, definindo os atributos e métodos
como públicos, privados ou protegidos, de acordo com as necessidades e boas práticas de programação.
- MVC (Model, View e Controller): Organize o projeto seguindo a arquitetura MVC, separando
adequadamente a lógica de negócio (Model), a interface de usuário (View) e a camada de controle
(Controller).
- Utilizar estruturas de laço e controle: Utilize estruturas de laço (for, foreach, while ou do while) e estruturas de
controle (if, else if, else ou switch) de maneira apropriada no sistema, proporcionando um fluxo lógico e
eficiente.
- Log: Crie um log que salve as informações do sistema, pode ser um log de erros ou de atividades que deve
ser salvo em txt.
- Serialização: Implemente o salvamento e recuperação dos objetos em arquivo.
- Clean Code: Aplique os princípios de Clean Code para escrever um código claro, organizado e eficiente,
facilitando sua manutenção e colaboração. Priorize nomes descritivos, estrutura modular, simplicidade e boas
práticas de formatação para garantir qualidade e legibilidade.
- Controle de Versão: Utilize o Git para controle de versão, com commits claros e organizados, feitos por todos
os membros da equipe ao longo do próximo mês.
